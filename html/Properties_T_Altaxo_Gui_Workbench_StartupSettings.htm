AllowAddInConfigurationAndExternalAddIns
Use the file ConfigDirectory\AddIns.xml to maintain a list of deactivated AddIns and list of AddIns to load from external locations. The default value is true.
AllowUserAddIns
Allow user AddIns stored in the "application data" directory. The default is false.
ApplicationName
ApplicationRootPath
Gets/Sets the application root path to use. Use null (default) to use the base directory of the application's AppDomain.
ConfigDirectory
Gets/Sets the directory used to store application properties, settings and user AddIns. Use null (default) to use "ApplicationData\ApplicationName"
DataDirectory
Sets the data directory used to load resources. Use null (default) to use the default path "ApplicationRootPath\data".
