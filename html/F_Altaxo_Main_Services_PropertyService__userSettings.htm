The user settings bag is a bag that loads the properties lazy (as Xml string) during creation of this service. See remarks for why this is neccessary.

Remarks
The properties in UserSettings are lazy loaded, i.e. the property values are stored as string first. Only if the property is needed, the xml string is converted into a regular value. This is neccessary, because some of the property values will required access to other property values during deserialization. But the user settings are deserialized in the constructor of the property service, and the property service is not installed in this moment. This will lead to NullReferenceException when the first deserialized property value try to get the property service.
